`GET /health`

Returns the current health status of the server. This endpoint does not require authentication and is used by load balancers, container orchestration platforms (such as Docker Compose and Kubernetes), and monitoring systems to confirm the service is alive.

<Note>
  This endpoint is registered at the root path, not under `/api/v1/auth`. The full URL is `http://your-host/health`.
</Note>

## Request

No parameters or authentication required.

## Response

<ResponseField name="status" type="string">
  Always `"ok"` when the server is healthy.
</ResponseField>

<ResponseField name="timestamp" type="number">
  Unix timestamp (milliseconds) at the time of the request. Useful for confirming the response is fresh.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl https://your-api.example.com/health
```

```typescript TypeScript
const response = await fetch('https://your-api.example.com/health');
const data = await response.json();
// { status: 'ok', timestamp: 1705312200000 }
```

</CodeGroup>

### Success response (200)

```json
{
  "status": "ok",
  "timestamp": 1705312200000
}
```

## Usage in Docker

The included `docker-compose.yml` uses this endpoint for its built-in health check:

```yaml
healthcheck:
  test: ["CMD", "node", "-e", "require('http').get('http://localhost:8080/health', (r) => process.exit(r.statusCode === 200 ? 0 : 1))"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 40s
```

See the [Docker deployment guide](/deployment/docker) for more details.
