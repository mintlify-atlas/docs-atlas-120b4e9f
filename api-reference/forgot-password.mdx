---
title: 'Forgot password'
description: 'Request a password reset email containing a link with a reset token for the given account.'
---

Trigger a password reset email for the specified address. The email contains a link with a reset token. Use that token in the `Authorization` header when calling [Reset password](/api-reference/reset-password).

<Note>
  For security reasons, this endpoint always returns a success response regardless of whether the email address is registered.
</Note>

## Request

**`POST /api/v1/auth/forgot-password`**

No authentication required.

### Body

<ParamField body="email" type="string" required>
  The email address of the account to reset the password for. Must be a valid email format.
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indicates whether the request succeeded. Always `true` on success.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="message" type="string">
      Confirmation message. Value: `"Password reset email sent"`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://your-api.com/api/v1/auth/forgot-password \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "user@example.com"
  }'
```

```typescript TypeScript
const response = await fetch('https://your-api.com/api/v1/auth/forgot-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'user@example.com' }),
});

const data = await response.json();
// { success: true, data: { message: 'Password reset email sent' } }
```

</CodeGroup>

### Success response

```json 200
{
  "success": true,
  "data": {
    "message": "Password reset email sent"
  }
}
```

### Error responses

```json 400
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid email address"
  }
}
```
